## User-Confirmed Project Doctrine

### Functional transfer over visual imitation

When reference fidelity conflicts with expressive independence, preserve the supported production function and change the concrete subject, composition, palette, typography, density, and travel mechanism.

Evidence: anti-clone project requirement, source observation, and round-two comparison.

### Readability over decorative density

When atmospheric detail conflicts with destination, promise, or action readability at thumbnail size, preserve the hierarchy and remove or isolate detail.

Evidence: round-one lower-copy failure and round-two footer repair.

### Inspectable production over opaque generation

When a visually plausible output cannot be revised or verified predictably, prefer an editable HTML/CSS source, deterministic rendering, and explicit full-size and thumbnail checks.

Evidence: project execution graph and generated source/render pair.

## Anti-Goals

- Near-copying a reference after changing only names or colors.
- Claiming source-creator philosophy from one artifact.
- Approving an image from prompt or source code without inspecting the render.
- Allowing the travel CTA to overpower destination desire.

## Scope And Confidence

- Scope: this Hokkaido poster test and similar auditable destination-poster experiments.
- Confidence: high for project doctrine because it is owner-confirmed and reflected in the evaluation repair; none for recovered creator doctrine.
