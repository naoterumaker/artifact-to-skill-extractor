# Method

## Project Doctrine

These are rules for this generated system, not recovered claims about the reference creator:

- Preserve production function while changing source expression materially.
- Preserve thumbnail readability when it conflicts with decorative density.
- Prefer an inspectable, editable source of truth over an opaque output when the task needs iteration.

## Composition Decision

```text
WHEN choosing anchor and density
IF immediate recognition outranks cultural breadth
DO use one dominant anchor with sparse supporting context
VERIFY destination identity survives at 240 px or less
ELSE use a controlled collage only when quiet fields preserve hierarchy
```

If either option remains too close to the reference arrangement, select a different spatial system or escalate bounded alternatives for human review.

## Design Contract

Choose one item from each function, then make their relationships explicit:

- **Identity anchor:** the first shape or subject recognized at thumbnail size
- **Context field:** environment that locates or qualifies the anchor
- **Directional device:** route, light, wind, shadow, or perspective that carries attention
- **Promise hierarchy:** destination or feeling first, useful promise second
- **Action:** departure, booking, visit, or exploration cue

The anchor and context must not compete. The directional device must lead through the promise toward action.

## Anti-Clone Transformation

Change all of the following unless the user has rights and asks for a close derivative:

1. subject family
2. spatial composition
3. palette and time of day
4. directional device
5. typographic alignment and scale relation
6. verbal promise and CTA mechanism

Changing nouns inside an otherwise identical scene graph is not enough.

## Render Checks

- Render the final dimensions; do not judge only from source code.
- Inspect a thumbnail no larger than 240 px wide.
- Confirm the identity anchor and main destination label survive reduction.
- Check that every text block has a quiet contrast field.
- Check all edges for clipping and all regions for unintended overlap.
- Compare side by side with the source and list retained functions versus changed expression.
