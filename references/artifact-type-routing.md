# Artifact Type Routing

Use the artifact type to decide what to inspect first. The abstraction target is always the production function, not the medium label.

## Finished Content

Examples: X posts, note articles, Zenn/Qiita posts, newsletters, YouTube videos, podcasts, LPs, ads, feed posts, books, decks, courses.

Inspect:

- Entry move: how the artifact earns attention or relevance
- Audience state: what prior belief, desire, objection, or confusion it assumes
- Sequence: why the order works
- Proof: evidence, examples, screenshots, demos, authority, specificity
- Transformation: what the reader/viewer can do or believe after consuming it
- CTA or next step: explicit or implicit action
- Medium constraints: length, visual density, playback, feed context, SEO, shareability

Common skill units:

- Content teardown workflow
- Offer or argument construction
- Proof selection rubric
- Hook/function translation across media
- Content quality rubric

## Process Trace

Examples: terminal log, browser session, edit history, coding session, design iteration, operational runbook, support transcript.

Inspect:

- Trigger condition: what started the workflow
- Inputs: files, systems, credentials, tools, assumptions
- Critical path: minimum sequence that must happen
- Branch points: decisions, retries, fallback routes
- Hidden checks: validations the expert performs without naming
- Error recovery: what happens when a tool or assumption fails
- Done condition: how completion is verified

Common skill units:

- SOP
- troubleshooting skill
- migration or deployment skill
- repository-specific workflow
- runbook-to-agent workflow

## Expert Conversation Or Interview

Examples: sales call, coaching session, consultation, meeting transcript, domain expert Q&A.

Inspect:

- Repeated distinctions: what the expert keeps separating
- Moment of correction: where the expert says "not that, this"
- Diagnostic questions: what they ask before acting
- Decision criteria: what makes one option better
- Boundary conditions: when they refuse, defer, or change approach
- Examples and counterexamples: what they use to teach judgment

Common skill units:

- diagnostic interviewer
- decision framework
- client intake skill
- feedback-to-rubric extractor
- tacit-knowledge codifier

## Code, Repository, Tool, API, CLI, Or Library

Inspect:

- Public capabilities and entry points
- Data model, commands, endpoints, flags, configuration
- Common workflows from README, docs, tests, and examples
- Error patterns and dependency assumptions
- Deterministic operations that should become scripts
- Fragile steps that require exact commands or schemas

Common skill units:

- tool helper skill
- API workflow skill
- repo-specific maintenance skill
- test/debug skill
- script-backed automation skill

## Book, Manual, Course, Or Body Of Knowledge

Inspect:

- Named frameworks, principles, methods, anti-patterns, and exact terms
- Chapter/module architecture
- Cases where the author applies the method
- Boundaries and critique
- Glossary and cross-links
- Practice prompts or transfer scenarios

Common skill units:

- framework skill
- methodology skill set
- glossary-backed reference skill
- chapter-on-demand knowledge base
- book-to-skill pipeline

## Comparative Set

Examples: winning vs losing ads, before/after drafts, multiple successful posts, rejected proposals, A/B tests.

Inspect:

- Invariants across winners
- Differences that correlate with failure
- Which surface traits are incidental
- Which functions recur through different expressions
- Rubric signals that explain quality
- Edge cases where a pattern stops working

Common skill units:

- quality rubric
- pattern library
- critique skill
- example-to-template workflow
- failure-mode detector

## Feedback Or Review Data

Examples: user corrections, PR reviews, client notes, edit requests, evaluation failures.

Inspect:

- Exact correction
- Underlying rule implied by the correction
- Scope: one artifact, one project, or reusable domain rule
- Test prompt that would have caught the failure
- Whether to update trigger, workflow, boundary, example, or rubric

Common skill units:

- skill update
- review checklist
- evaluator
- regression test pack
