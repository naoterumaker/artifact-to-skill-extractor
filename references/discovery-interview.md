# Discovery Interview

Use the interview to define the capability contract, not to make the user repeat information already visible in artifacts.

## Interview Order

1. Inspect supplied artifacts, notes, links, analytics, and existing skills.
2. State the capability you infer in one sentence.
3. Propose likely inputs, outputs, quality goals, and research axes.
4. Ask only about unresolved choices that materially change the skill.
5. Write the agreed contract using `assets/templates/capability-contract.yaml`.

## Core Questions

Ask no more than necessary from these groups:

| Group | Question |
|---|---|
| Outcome | What should the future agent reliably make possible? |
| Profile | Is this mainly procedural, creative, or tool-dependent hybrid work? |
| User | Who invokes it, and what can be assumed about them? |
| Trigger | In what situation should the skill activate? |
| Inputs | What evidence, files, links, data, or context will normally be available? |
| Output | What exact artifact or action constitutes completion? |
| Doctrine | Which value conflicts, evidence standards, audience relationship, and anti-goals govern choices? |
| Authority | Which decisions may the agent make, and which require approval? |
| Quality | What would make the user say the result is excellent rather than merely acceptable? |
| Distribution | Where is the output consumed, measured, updated, reused, or retired? |
| Boundary | What must never be copied, claimed, executed, or changed? |

## Creative-Output Extension

When the skill creates content, separate these questions:

- **Modality**: text, image, video, audio, or mixed
- **Delivery platform**: X, YouTube, note, LP, email, etc.
- **Primary objective**: impressions, CTR, retention, engagement, conversion, education, trust
- **Audience state**: current belief, desired belief/action, prior relationship
- **Account context**: existing audience, successful/failed content, available analytics
- **Creative constitution**: point of view, emotional temperature, proof standard, taste exclusions
- **Source of truth**: prose, code, canvas, timeline, project file, or remote system
- **Agency**: which taste, rights, publication, and irreversible decisions require a human

Do not ask for a generic "good post" or "good video." Convert quality into an objective and observable criteria.

## Research-Specific Extension

For research skills, clarify:

- decision or deliverable the research supports
- freshness requirements
- acceptable and prohibited source types
- geographic/language scope
- evidence and citation standard
- contradiction policy
- stopping rule

## Completion Check

The contract is ready when a different agent can answer:

1. When do I activate?
2. What do I receive?
3. What do I produce?
4. What decisions can I make?
5. How will my output be judged?
6. What must I not do?
