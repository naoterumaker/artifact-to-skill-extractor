# Generator Protocol

The Generator compiles the supported production-system layers into an agent-callable package and produces a realistic output by executing that package.

## Inputs

- approved capability contract
- evidence manifest and observations
- comparison matrix and evidence ceiling
- supported doctrine, grammar, and decision units
- capability registry and execution graph when required
- verified or clearly labeled hypothesis patterns
- frozen evaluation rubric
- relevant modality, platform, and objective references

## Generation Order

1. Confirm which production-system layers are evidenced and which are explicitly absent.
2. Validate decision-unit interfaces and capability coverage.
3. Create the system and skill specification.
4. Choose the minimum file structure for the selected profile.
5. Write activation conditions and near-miss boundaries.
6. Compile decision routing, execution graph, verifiers, fallbacks, and approvals.
7. Place doctrine, grammar, detailed knowledge, and examples in references.
8. Place copyable output templates in assets.
9. Add deterministic scripts or adapters only where judgment is not required.
10. Generate trigger, functional, prediction, recovery, and boundary tests.
11. Execute the compiled system on at least one unseen test input.

## Independence Rules

- Do not copy source wording or composition merely to raise similarity.
- Do not inspect the Evaluator's final diagnosis before producing the first output.
- The Generator may see the frozen rubric because it defines the target.
- Do not alter the rubric during generation.
- Do not silently invent creator doctrine when only project doctrine is available.
- Do not select a tool before required capabilities and constraints are defined.

## Output Trace

Every important generated rule should record one of:

- source observation id
- user-confirmed requirement
- research evidence id
- explicit hypothesis label

Decision units must also retain scope, confidence, counterevidence, verifier, fallback, and agency assignment.

If a rule cannot be traced, omit it or mark it as a hypothesis.
