# Self-Extraction Audit

## Artifact

The input artifact is this repository itself: `SKILL.md`, reference routing, templates, scripts, tests, and worked outputs.

## Recovered Production Function

The repository is not merely an extraction prompt. Its production function is to convert incomplete evidence into a scoped, testable capability through a closed harness:

```text
capability contract
  -> evidence plan
  -> artifact-native observation
  -> evidence-labeled abstraction
  -> frozen rubric
  -> generated skill
  -> unseen test output
  -> evaluation and root-layer repair
  -> scoped lifecycle decision
```

The durable logic belongs in `SKILL.md` and core references. Concrete source fixtures, renders, and evaluation records belong in `assets/examples/`. Deterministic structure, arithmetic, estimation, and fixture checks belong in `scripts/`. Mutable client run data belongs in an external project workspace.

## Round 1 Finding

Applying the harness to itself exposed a meta-level gap: the per-run Generator/Evaluator loop was present, but there was no consistent protocol for cross-domain promotion, repeated-failure harvesting, lifecycle demotion, or large-corpus estimation.

The retrospective self-audit scored the pre-repair state at `75.70`, with `self-application-and-limits` below the critical minimum. The repair was made at the reference architecture and deterministic support layers rather than by adding more prose to the generated examples.

Added:

- `references/transfer-protocol.md`
- `references/failure-harvest-loop.md`
- `references/lifecycle.md`
- `references/lifecycle-status.yaml`
- `references/cost-aware-extraction.md`
- `scripts/estimate_extraction.py`
- `scripts/run_test_prompts.py`

## Round 2 Decision

Round 2 scores `88.20` and passes the structural self-audit. The detailed rubric and results are under `assets/examples/self-extraction/`. Because this rubric was reconstructed after initial v0.2 implementation began, the audit is diagnostic evidence, not valid pre-registered promotion evidence.

This does not justify `active` status yet. The X and image tests each supply only one forward case, and the evaluator is the implementing agent. The lifecycle record therefore remains `experimental` and explicitly lists untested modalities.

## Why This Is Self-Application

The skill did not only describe itself. It:

1. treated its own repository as evidence
2. reconstructed its production function and resource boundaries
3. froze a rubric for the metaskill
4. found a failing critical dimension
5. routed the repair to missing reference and script layers
6. re-evaluated and preserved unresolved limits

The same standard used for external artifacts therefore changes the metaskill itself.
