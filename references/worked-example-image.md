# Worked Example: Image Hybrid System

## Test Question

Can one licensed image be observed in a structured form, abstracted beyond that serialization, compiled with explicit doctrine and tool capabilities, and used to create a recognizably different image?

## Evidence, Rights, And Ceiling

The source is GDJ's `Vintage Travel Poster Paris`, retrieved from Wikimedia Commons with an Open Clip Art origin and a CC0 1.0 Universal dedication. The local raster and source manifest are:

- `assets/examples/image/source-vintage-travel-poster-paris.png`
- `assets/examples/image/source-manifest.yaml`

CC0 permits copying, modification, and redistribution to the extent allowed by law. The example still credits the source and applies an anti-clone gate.

One poster supports observed spatial structure and candidate visual grammar. It does not establish the original creator's philosophy, all destination-poster principles, or audience effectiveness.

## Native Representation And Abstraction

Image geometry is represented in `assets/examples/image/observation.yaml`: regions, entities, spatial relations, hierarchy, palette, and typography. Meaning, taste, and inferred intent remain in `assets/examples/image/analysis.md`, where uncertainty can be expressed directly.

The observation YAML is a concrete serialization of one poster, not the abstraction. Source-specific expression includes the Paris wording, central draped figure, split field, sun face, airplane, architecture, produce, and typography.

The candidate functional relation in `assets/examples/image/pattern.yaml` is:

```text
dominant destination identity
  + subordinate place context
  + integrated naming
  + directional access
  -> recognizable promise
  -> feasible action
```

Variables include anchor, context, directional device, verbal sequence, density, and palette. Constraints and selection rules prevent arbitrary recombination.

## Compiled Hybrid System

The v0.3 intermediate representation is under `assets/examples/image/creative-system/`.

| Layer | Compiled result | Evidence status |
|---|---|---|
| Profile | `hybrid` | capability and environment decision |
| Recovered creator doctrine | explicitly unavailable | insufficient evidence |
| Project doctrine | functional transfer over imitation; readability over decoration; inspectable production over opaque generation | user-confirmed |
| Grammar | destination anchor, supporting context, route cue, verbal hierarchy, conditional density | candidate |
| Decision unit | select anchor and density from target recognition, breadth, quiet fields, and anti-clone conditions | experimental |
| Capability registry | parameterized layout, deterministic browser render, thumbnail generation | locally verified |
| Execution graph | observe, select, author, full render, thumbnail, evaluate, bounded repair | complete structural contract |

The tool registry records interface, modality, edit granularity, source of truth, state visibility, partial editability, reproducibility, parameterization, batch support, preview, evaluation surface, human round-trip, constraints, license, recovery, version, and verification date. Playwright and SIPS are selected because they satisfy required capabilities, not because they are generally preferred tools.

## Generated Skill And New Concrete

The candidate skill is `assets/examples/image/destination-poster-recomposer/SKILL.md`. Its test output is a contemporary Hokkaido morning-train poster. It changes destination, transport, scene graph, density, palette, typography, directional mechanism, wording, and promise.

The editable source is `assets/examples/image/poster-round-2.html`; the rendered output is `assets/examples/image/poster-round-2.png`; thumbnail inspection uses `assets/examples/image/poster-round-2-thumbnail.png`.

## Evaluator Loop

The rubric was frozen at `assets/examples/image/evaluation-rubric.json`.

| Round | Score | Decision | Root issue |
|---|---:|---|---|
| 1 | 75.00 | revise | low-contrast lower copy failed the critical readability dimension |
| 2 | 88.20 | pass | no blocking issue; audience and campaign performance remain untested |

Round 1 repaired both the permanent method and the current output: the skill gained mandatory full-size and thumbnail checks with a quiet-field rule, while the poster gained a dedicated footer hierarchy.

## What This Establishes

- YAML can preserve exact image observations and handoff fields.
- A second abstraction step is required to produce grammar, variables, constraints, and conditional decisions.
- Project doctrine is distinct from inferred source-creator doctrine.
- Tool choice changes the source of truth, editability, reproducibility, and evaluation method.
- Rendered pixels at target and thumbnail sizes are required evidence; HTML, prompts, or YAML alone are insufficient.
- A new concrete can retain production function while changing source expression materially.

The system remains experimental because the corpus contains one reference poster, one forward output, no independent evaluator, and no audience or campaign outcome data.
